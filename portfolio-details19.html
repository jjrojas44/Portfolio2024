Terminator: Future war

The UI for the terminator: resistance sequel,Terminator: Future war.

Case Study:

Terminator: Future war
Project Overview
  • Project Name: Terminator: Future war (Hypothetical Sequel)
  • Role: UX/UI Designer (Solo Designer)
  • Tools Used: Figma,Photoshop,Photopea,Gameuidatabase,dribbble,Behance,
  • Timeline: 3 Weeks
  • Objective: Introduce a hypothetical UI concept for a sequel called Terminator: Future War.
Value Added

The Terminator: Future War UI concept adds value by creating an immersive, retro-futuristic interface that enhances both gameplay and storytelling.

Problem Statement

Challenges with the Existing UI from Terminator: Resistance:

  • Cluttered HUD and Menus: Players found it difficult to quickly access mission-critical information and objectives during fast-paced combat situations.
  • Weak Thematic Integration: The UI lacked cohesion with the gritty, dystopian tech-driven aesthetic of the Terminator universe, breaking immersion.
  • Poor Visual Hierarchy: Essential gameplay data was often buried or visually overshadowed by less relevant UI elements, reducing usability under pressure.
  • Limited Accessibility Options: The game offered minimal options for HUD scaling, colorblind modes, or UI customization, limiting usability for a diverse player base.

Why a Design for a Hypothetical Sequel?

  • To create a cinematic, immersive, and faction-specific interface that supports storytelling and strengthens narrative worldbuilding.
  • To enhance overall usability with streamlined menus, mission trackers, and clear tactical overlays for improved gameplay flow.
  • To modernize the interface while staying true to the dystopian, war-torn visual identity of the Terminator universe, embracing Skynet-era design language.

Research

Persona

User Insight

  • Players consistently described Terminator: Resistance’s UI as cluttered and unintuitive, especially during combat-heavy scenarios where clarity was vital.
  • There was a strong desire for a more immersive, retro-futuristic digital aesthetic evocative of Skynet's oppressive world.
  • Gamers preferred streamlined, context-based HUD elements, modular inventory systems, and quick-access menus over static or layered interfaces.
  • A noticeable frustration came from the lack of optimization for both console and mobile controls when managing inventories and navigating menus.

POV Statement

The user interface I want should be clean, minimal, and customizable, enhancing gameplay without breaking immersion so I can stay fully engaged in both the combat and story-driven moments without distraction.

How Might We

  • How might we design a clean, futuristic, and faction-based HUD that maintains immersion and elevates storytelling during high-stakes combat?
  • How might we simplify menu structures and layouts for better accessibility across both PC and console platforms?
  • How might we faithfully integrate Terminator’s dystopian visual language into the UI without compromising usability?
  • How might we improve inventory, mission tracking, and status indicators through modular, context-sensitive layouts?

Heuristic Evaluation

Purpose: Validate UI concepts against established design heuristics while ensuring consistent alignment with the Terminator lore and Skynet-era visuals.

Sources: Referenced UI systems from Terminator: Dark Fate, Terminator: Salvation, Terminator: Resistance, T-800 HUD film footage, and benchmarked against other shooter games like Division 2, Metro Exodus, and Black Ops 6.

Outcome: Produced a franchise-consistent, modular UI style guide for the hypothetical Future War sequel.

Evaluation Summary

  • Visibility of System Status: Metro Exodus and Division 2 excelled at situational awareness cues. Terminator’s UI lacked real-time, clear threat indicators.
  • Match Between System & Real World: Metro’s in-world UI was deeply immersive, while BO6 sometimes strayed too abstract. Terminator failed to align with its narrative world.
  • User Control & Freedom: Division 2 offered strong UI customization, Terminator was too rigid with fixed layouts.
  • Consistency & Standards: BO6 and Division 2 followed strong genre conventions; Terminator’s interface broke consistency with genre and lore standards.
  • Error Prevention: Division 2 prevented common errors (like accidental gear loss). Terminator lacked safeguards like save warnings.
  • Recognition vs. Recall: BO6 and Division 2 effectively used iconography and visual cues; Terminator buried information in dense menus.
  • Flexibility & Efficiency: Division 2 provided scalable HUD options; Terminator’s UI remained static and inflexible.
  • Aesthetic & Minimalist Design: Metro’s minimal HUD faded during exploration for immersion. BO6 cluttered mid-combat. Terminator lacked modern minimalism.
  • Error Recovery: Division 2 offered clear error messages and context-specific help. Terminator defaulted to generic error text.
  • Help & Documentation: Division 2 integrated contextual tutorials; Metro lacked menu guidance. Terminator’s help systems were sparse and outdated.

Key Takeaways

Strengths: Division 2 leads in UI customization and error prevention. Metro delivers unmatched immersive in-world UI. BO6 retains familiar controls and readability.

Weaknesses: Terminator: Resistance suffers from rigid HUD layouts, inconsistent immersion, and poor error management. BO6 struggles with combat UI clutter.

Opportunities: Add scalable, faction-themed HUD presets in Terminator. Improve contextual tutorials in Metro. Provide an immersive, minimal HUD mode for BO6. Use Division 2’s modular UI system as a benchmark framework.
